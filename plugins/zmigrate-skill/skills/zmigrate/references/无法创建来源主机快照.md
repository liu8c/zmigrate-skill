复制任务的X-ray

Linux来源主机的X-ray

client：来源主机的X-ray。
	•	jobs：存放这台来源主机的job信息。
	•	logs：存放来源主机的日志信息。
server：包含中转TARServer和目标RK（部分架构中，目标可能是TARServer）。
	•	jobs：有关这台TARServer中的所有job信息。
	•	logs：有关这台TARServer中的所有log信息。

Windows
使用VSS
#查看vss快照 vssadmin list shadows 

尝试手动拍摄快照
	•	任选一个盘符，右击选择“配置卷影副本”。

	•	依次对各盘符手动拍快照，成功拍出快照，截图为手动成功拍出C盘的快照，快照在3的位置。

	•	如果每个分区都能成功拍出快照，则对复制任务发起同步。
	•	如发起同步，依旧报无法创建快照，可能是磁盘I/O较差，拍摄快照超时。
	•	如果有1个或者多个分区手动拍快照失败，则修改卷影副本配置。
	•	依次选中各盘符，修改设置

	•	在使用限制和没有限制之间切换。
	•	已勾选没有限制，则切换至使用限制，确定并保存。
	•	已勾选使用限制，则切换至没有限制，确定并保存。

	•	重复上面的操作，再次尝试手动拍快照。
	•	以上方法均无法解决无法拍快照的问题，则使用non-VSS模式。
使用non-VSS
前提：在设定中把UI设定为“除错”。

在创建复制任务，或者编辑复制任务时，关闭“快照代理”。
提示：
可以在一个复制任务中，反复切换VSS与non-VSS。
复制任务的配置信息，修改后再下一次同步时生效。

Linux
空间不足
默认配置，要求每个挂载点有10%的空闲空间。

ext3导致超时
ext3是较老的文件类型，拍快照的时候，需要把快照空间.snapshot0重写zero，会耗费较长的时间，具体时间取决于磁盘的I/O。

对于特别大的磁盘ext3需要修改ratio。
挂载点太多
默认支持48个挂载点，超出48个挂载点需要特殊处理。

磁盘类型不正确

Linux修改ratio

	•	ratio的值是百分比，10表示10%。
	•	max_size的单位是M，0表示不限制，500表示snapshot的最大值为500M。
	•	创建的snapshot≥500M，则限制snapshot=500M。
	•	创建的snapshot＜500M，则限制snapshot取实际值。
